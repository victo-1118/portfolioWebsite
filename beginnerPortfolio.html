User Authentication System (Backend)

Purpose

I started this project to learn more about FastAPI and how I could use a database with it

Technologies used

  • FastAPI: for building the web API
  • Tortoise ORM: for making the database and its interactions
  • Python: The primary programming language
  • JWT (Json Web Tokens): Used to implement secure authentication

Role

My primary role was to study and understand the existing code. I did not write any of the code but focused on dissecting every part

Learning process

  1. Understanding FastAPI: I explored the basics of FastAPI, including setting up endpoints, handling requests and responses and using dependency injection
  2. Understanding Tortoise ORM: I learned how to establish my database connection , create models, used Tortoise ORM methods to help implement crud operations
  3. Implemented JWT: I learned about JWT tokens and how it is used for secure authentication including generation and verifying tokens

Challenges

Challenge: Understanding the flow of user authentication and how different components interacted

Solution: I broke down the code into smaller manageable pieces to understand what each part accomplished I also looked at tutorials and used Technologies like chat gpt to help me understand the reason behind some the code

Challenge: Ensuring best security practices

Solution: I replaced any deprecated code with up to date methods

Learning and Outcomes

  • Skills Acquired: Gained a better understanding of FastAPI and Tortoise ORM. Also Improved my knowledge of web security principles and authentication mechanisms.
  • Insights: Understood I needed to learn more about file structure when creating RESTful API's . Best security practices is also something I need to work on as I realize theres a lot to learn about that still.

To Do List API (Frontend and Backend)

Purpose

My purpose for this project was to implement my better understanding of FastApi and Tortoise ORM

Technologies Used

  • FastAPI: for building the web API
  • Tortoise ORM: for making the database and its interactions
  • Python: The primary programming language
  • Typing library: Used for type hinting which improve quality, readability and maintainability of the code

Role

My primary role was to code the frontend and backend part of the project to solidify my understanding of FastAPI, Tortoise ORM and (frontend stuff)

Learning Process

  1. Continued further Understanding of FastAPI and Tortoise ORM: Implemented my what I understood from both of these technologies and continued to learn more features from them
  2. Implemented and Understanding Typehinting in Python: Learned about Typehinting to begin to develop good habits and to make

Challenges

Challenge: How to implement a class that would act as a list that would take several Item objects

Solution: I learned that the fields library has a two methods that can help simulate a list. By using .ReverseRelation() and .ForeignKeyField() I was able to establish a relationship between the two classes(List and Item) so that I could link Item objects to a List object which would be the only object that could access those Item objects.

Challenge: How to use pydantic models made from Tortoise ORM for typehinting

Solution: I went to the Tortoise ORM documentation for help and saw that I just had to create classes based on the variable names of my pydantic models created from Tortoise and pass them so that I could use them for type checking.

Learning and Outcomes

  • Skills Acquired: Learned how to simulate list and how to use pydantic models created from Tortoise for typehinting. Also gained practical experience on how to create RESTful API's, how I should set up my file structure and what order I should work for parts of the API
  • Insights: Understood that theres still a lot for me to learn about creating automatic tests.
